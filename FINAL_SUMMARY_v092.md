## 🔍 技术备注

### CSS 编译工具
- **工具**：lessc（已在 WSL 中安装）
- **命令**：
  ```bash
  wsl -d Ubuntu-24.04 -u root lessc <input.less> <output.css>
  ```
- **编译速度**：毫秒级

### 部署方式
1. **自动部署**：`wsl -d Ubuntu-24.04 -u root bash deploy_css.sh`
2. **手动部署**：逐个 `cp` 文件到 `/usr/local/freecdn/edge-admin/`
3. **验证**：`ls -lh /usr/local/freecdn/edge-admin/web/views/@default/`

## 🎓 学到的经验

### ✨ 最佳实践
1. **CSS 架构**：使用变量和 mixin 提高可维护性
2. **版本管理**：保持 code const 与 Release tag 的一致性
3. **部署自动化**：编写脚本减少手工操作
4. **文档完整性**：为每个版本撰写详细的发布说明

### ⚠️ 需要注意的坑
1. **CRLF 换行符**：Windows 编辑的 `.sh` 文件需要转换为 LF
2. **编码问题**：Python 脚本在 GBK 编码系统上需要处理 UTF-8
3. **浏览器缓存**：CSS 更新后必须清缓存
4. **WSL 权限**：需要 root 权限才能写入 `/usr/local/`

### 💡 改进建议
1. **CI/CD 流程**：自动编译 LESS 和部署 CSS
2. **测试框架**：为前端加入单元测试
3. **设计系统**：建立完整的 UI 组件库文档
4. **监控告警**：部署后自动检查服务状态
