fix: set create_system=3 in zipfile to fix directory permissions on Linux

Python zipfile on Windows defaults create_system=0 (Windows), causing Go's
archive/zip to ignore Unix permission bits and create dirs with 0664 (no x bit),
leading to 'permission denied' when accessing subdirectories. Fix: explicitly
set create_system=3 (Unix) on all ZipInfo entries in _make_node_zip().
