# FreeCDN 架构说明

## 系统概览

FreeCDN 基于 GoEdge v1.3.9 二次开发，保留三层架构设计：管理台（EdgeAdmin）、配置枢纽（EdgeAPI）、边缘节点（EdgeNode）。三个组件通过 gRPC 通信，数据持久化到 MySQL。

```
┌─────────────────────────────────────────────────────────┐
│                    FreeCDN 系统架构                       │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  ┌──────────────┐    gRPC/8003    ┌──────────────────┐  │
│  │  EdgeAdmin   │◄───────────────►│    EdgeAPI       │  │
│  │  (Web 管理台) │                 │  (配置枢纽节点)   │  │
│  │  Port: 7788  │                 │  Port: 8003      │  │
│  └──────────────┘                 └────────┬─────────┘  │
│                                            │             │
│                                   ┌────────▼─────────┐  │
│                                   │    MySQL 8.0     │  │
│                                   │  (配置+统计数据)  │  │
│                                   └──────────────────┘  │
│                                                          │
├─────────────────────────────────────────────────────────┤
│                      边缘节点层                           │
│                                                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │  EdgeNode 1  │  │  EdgeNode 2  │  │  EdgeNode N  │  │
│  │  (amd64)     │  │  (arm64)     │  │  (任意架构)   │  │
│  │  Port: 80/443│  │  Port: 80/443│  │  Port: 80/443│  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
│                                                          │
└─────────────────────────────────────────────────────────┘

用户请求 → DNS 解析到最近 EdgeNode → 命中缓存直接返回
         → 未命中 → WAF 检查 → 回源 → 缓存 → 返回
```

## 通信协议

### EdgeAdmin ↔ EdgeAPI

EdgeAdmin 通过 gRPC（默认端口 8003）与 EdgeAPI 通信，所有配置变更（域名、节点、证书、规则等）都通过此通道写入数据库。双方使用对称密钥（`secret`）做 gRPC 认证。

### EdgeAPI → EdgeNode

EdgeNode 启动后主动连接 EdgeAPI（配置的 `endpoints` 地址），建立长连接。EdgeAPI 通过此长连接推送配置变更，EdgeNode 实时热加载，无需重启。

### EdgeNode 数据上报

EdgeNode 定期将访问统计、节点状态等数据回传 EdgeAPI，写入 MySQL，供 EdgeAdmin 展示图表。

## 数据流

### CDN 请求处理流程

```
1. 用户 → DNS 解析 → EdgeNode IP
2. EdgeNode 检查本地缓存
   ├── 命中 → 直接返回（cache hit）
   └── 未命中 →
       3. WAF 规则检查（IP 黑名单、CC 频率、规则引擎）
          ├── 拦截 → 返回 403/444
          └── 通过 →
              4. 回源请求（HTTP/HTTPS/TCP/UDP）
              5. 缓存响应（按配置的 TTL）
              6. 返回给用户
```

### 配置变更流程

```
1. 管理员在 EdgeAdmin 操作（如：修改域名配置）
2. EdgeAdmin 通过 gRPC 调用 EdgeAPI 写入 MySQL
3. EdgeAPI 检测到配置变更
4. EdgeAPI 通过长连接推送变更给所有在线 EdgeNode
5. EdgeNode 热加载新配置（无需重启）
```

## 部署拓扑

### 最小部署（单机）

适合个人测试或小流量场景：

```
┌─────────────────────────────┐
│       单台服务器（2核2G）     │
│                             │
│  EdgeAdmin（:7788）         │
│  EdgeAPI   （:8003）        │
│  EdgeNode  （:80/:443）     │
│  MySQL     （:3306）        │
│                             │
└─────────────────────────────┘
```

注意：单机部署时管理台和边缘节点在同一服务器，适合轻量使用。EdgeNode 监听 80/443，EdgeAdmin 监听 7788（建议防火墙只对内网开放）。

## 端口规划

| 端口 | 服务 | 说明 |
|------|------|------|
| 7788 | EdgeAdmin HTTP | 管理后台，建议仅内网访问 |
| 8003 | EdgeAPI gRPC | EdgeAdmin 和 EdgeNode 均需访问此端口 |
| 80   | EdgeNode HTTP | 边缘节点 CDN 流量 |
| 443  | EdgeNode HTTPS | 边缘节点 CDN 流量（TLS） |
| 3306 | MySQL | 仅管理节点本地使用，不对外暴露 |

## 安全设计

### 认证机制

- EdgeAdmin ↔ EdgeAPI：对称密钥认证（gRPC header `secret`）
- EdgeAPI ↔ EdgeNode：节点注册时颁发唯一 `nodeId` + `secret`，长连接持续验证
- 管理台登录：用户名 + 密码，支持防暴力破解

### 推荐加固措施

1. 防火墙仅对 EdgeNode 开放 80/443，7788 和 8003 只允许信任 IP 访问
2. MySQL 不对外暴露（3306 不开放外部访问）
3. 管理台域名配置 HTTPS（EdgeAdmin 后面挂 Nginx 反代 + Let's Encrypt）
4. 甲骨文云需同时配置 iptables 和 OCI 安全列表

## 版本治理模型

FreeCDN 的治理方向是“上游仅作为安全审计输入，FreeCDN 作为唯一发布输出”。

```
上游变更情报（仅输入）
        │
        ▼
安全审计与差异评估
        │
        ▼
FreeCDN 内部实现 / 选择性合并
        │
        ▼
FreeCDN 版本发布（唯一输出）
```

治理要点：

1. 不自动同步上游版本和发布物。
2. 外部变更必须通过审计与验证后才可进入代码库。
3. 对外只提供 FreeCDN 自有安装链路与发布产物。
4. 发布与回滚流程统一遵循 [FAQ 的版本治理与发布](./FAQ.md#版本治理与发布)。
